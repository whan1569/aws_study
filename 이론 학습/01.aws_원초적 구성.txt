
원초적 구성:

1. 데이터 센터 (Data Centers)
   - AWS 클라우드는 전 세계 여러 지역에 위치한 데이터 센터에 기반을 두고 운영됩니다. 각 데이터 센터는 고속 네트워크와 전원 시스템, 보안 시스템 등을 갖추고 있으며, 여러 개의 가용 영역 (Availability Zones, AZs)으로 나누어져 있습니다. 각 AZ는 서로 물리적으로 분리되어 있지만, 고속 네트워크로 연결되어 있습니다.

2. 리전 (Region)
   - AWS는 여러 리전 (Region)으로 나누어져 있습니다. 각 리전은 특정 지리적 위치에 위치한 데이터 센터들의 집합으로, 여러 개의 가용 영역을 포함하고 있습니다. 예를 들어, US-East-1 (버지니아 북부) 또는 EU-West-1 (아일랜드) 등의 리전이 있습니다.

3. 가용 영역 (Availability Zones, AZ)
   - 각 리전 내에는 2개 이상의 가용 영역이 있으며, 이는 데이터 센터가 물리적으로 분리되어 있기 때문에 리전 내에서 고가용성을 유지할 수 있습니다. 가용 영역들은 동일한 리전 내에서 네트워크로 연결되며, 하나의 AZ가 장애를 겪더라도 다른 AZ에서 서비스가 계속 운영되도록 설계되어 있습니다.

4. 엣지 로케이션 (Edge Locations)
   - AWS는 CloudFront와 같은 콘텐츠 전송 네트워크(CDN) 서비스의 핵심으로 사용되는 엣지 로케이션을 운영합니다. 엣지 로케이션은 리전과 가용 영역과는 별개로, 전 세계 여러 위치에 분포하며, 사용자와의 지리적 거리를 줄여 빠른 콘텐츠 전송을 지원합니다.
