1. **IP 주소**
AWS Lightsail 인스턴스는 **고정 IP (Static IP)**와 **동적 IP (Dynamic IP)**를 제공합니다.

1.1. **동적 IP (Dynamic IP)**
- **기본 IP**: AWS Lightsail 인스턴스에 생성 시 자동으로 할당되는 **동적 IP**는 인스턴스를 종료하거나 재시작하면 변경될 수 있습니다. 
- **사용 시 주의**: 서버가 다운되거나 다시 시작되면 IP 주소가 바뀔 수 있기 때문에, 고정된 IP를 사용해야 할 경우 고정 IP를 할당받는 것이 좋습니다.

1.2. **고정 IP (Static IP)**
- **고정 IP**는 인스턴스를 시작할 때 변경되지 않는 **고정된 IP** 주소를 제공합니다.
- **고정 IP 할당**: Lightsail에서 고정 IP를 할당하려면, **고정 IP**를 인스턴스에 연결해야 합니다. 이를 통해 인스턴스가 종료되거나 재시작되어도 IP 주소가 바뀌지 않습니다.
  
  **고정 IP 할당 절차**:
  1. Lightsail 콘솔에서 **Networking** 탭을 클릭합니다.
  2. **Static IP** 섹션에서 **Create Static IP**를 클릭합니다.
  3. 해당 IP를 연결할 인스턴스를 선택하고 **Create**를 클릭하여 고정 IP를 할당합니다.

2. **방화벽 (Firewall) 설정**
AWS Lightsail의 방화벽은 **인스턴스에 대한 네트워크 접근을 제어**하는 기능을 제공합니다. **포트**와 **프로토콜**을 기반으로 네트워크 트래픽을 필터링할 수 있습니다.

2.1. **기본 보안 규칙**
- Lightsail은 기본적으로 모든 인스턴스에 대해 **퍼블릭 IP**를 할당하고, 기본적으로 **22번 포트(SSH)**와 **80번 포트(HTTP)**만 개방되어 있습니다.
  
2.2. **새로운 포트 열기**
- 인스턴스에 접속하려면 필요한 포트를 열어야 합니다.
  1. Lightsail 콘솔에서 **Networking** 탭으로 이동합니다.
  2. **Firewall** 섹션에서 **Add another**를 클릭합니다.
  3. 열고자 하는 포트(예: **3306**(MySQL), **443**(HTTPS))와 **Source type**을 설정합니다.
    - `Anywhere`: 모든 IP 주소에서 접근 가능
    - `My IP`: 내 IP 주소에서만 접근 가능
    - `Custom`: 특정 IP에서만 접근 가능

2.3. **보안 규칙 예시**
- **SSH (포트 22)**: 인스턴스에 로그인하기 위한 SSH 연결을 허용하는 포트입니다.
- **HTTP (포트 80)**: 웹 서버에서 HTTP 프로토콜을 통해 웹 애플리케이션에 접속할 수 있게 하는 포트입니다.
- **HTTPS (포트 443)**: 보안 연결을 위한 HTTPS 프로토콜을 사용하는 포트입니다.
- **MySQL (포트 3306)**: MySQL 데이터베이스에 접근하기 위한 포트입니다.
- **Custom Ports**: 자신의 필요에 맞는 포트를 추가할 수 있습니다.

3. **VPC (Virtual Private Cloud) 설정**
VPC는 **가상 네트워크**를 설정하고, 여러 인스턴스와 리소스들이 어떻게 상호작용하는지 정의하는 기능입니다. Lightsail에서는 기본적으로 VPC를 사용하여 네트워크를 구성합니다.

3.1. **Lightsail VPC**
- **Lightsail 기본 VPC**: Lightsail 인스턴스는 자동으로 기본 VPC에 배치됩니다. 이 VPC는 자체적인 **서브넷**, **라우팅 테이블**, **인터넷 게이트웨이**를 제공합니다.
- **VPC 커스터마이징**: Lightsail에서 기본 제공하는 VPC 외에 **AWS VPC**를 이용해 더 복잡한 네트워크 구성을 할 수 있습니다.

3.2. **VPC의 주요 구성 요소**
- **서브넷(Subnet)**: 인스턴스와 리소스를 배치할 수 있는 네트워크 구역입니다.
- **인터넷 게이트웨이(IGW)**: VPC의 서브넷과 외부 인터넷 간의 연결을 가능하게 하는 리소스입니다.
- **라우팅 테이블(Route Table)**: VPC 내의 네트워크 트래픽을 제어하는 표입니다.

4. **DNS 설정**
Lightsail에서는 도메인 이름을 사용하여 인스턴스를 쉽게 식별할 수 있습니다. **Lightsail DNS**를 사용하여 도메인 이름을 관리하고, **Elastic IP**나 고정 IP에 연결할 수 있습니다.

4.1. **Lightsail DNS 사용하기**
- Lightsail에서는 DNS 레코드를 **A 레코드**, **CNAME 레코드** 등으로 설정하여 도메인 이름과 고정 IP를 연결할 수 있습니다.
- **A 레코드**는 도메인 이름을 고정 IP와 연결하는 데 사용됩니다.
  1. Lightsail 콘솔에서 **Networking** 탭을 클릭합니다.
  2. **DNS** 섹션에서 **Create DNS zone**을 클릭하고, 도메인 이름을 추가합니다.
  3. 도메인 이름에 해당하는 **A 레코드**를 추가하여 인스턴스의 고정 IP를 연결합니다.

4.2. **도메인 이름 등록**
- Lightsail에서는 자체적으로 도메인 이름을 등록할 수 있습니다. **Lightsail DNS**를 통해 도메인을 구매하고, DNS 레코드를 관리할 수 있습니다.
  1. **Create DNS Zone**을 클릭하여 도메인을 생성합니다.
  2. 도메인에 대한 DNS 레코드를 추가하여 웹 애플리케이션을 쉽게 접근할 수 있도록 설정합니다.

5. **Private IP 주소**
Lightsail은 **Private IP**도 제공하여, 외부 인터넷과 연결되지 않고 인스턴스들 간에 내부 통신이 가능합니다. 내부 네트워크에서만 접근 가능한 서비스를 설정할 때 유용합니다.

5.1. **Private IP 사용**
- Private IP를 이용하면 **VPC** 내에서만 통신하는 인스턴스를 만들 수 있습니다. 예를 들어, 데이터베이스 인스턴스를 외부와 연결하지 않고, 다른 인스턴스만 연결하려면 Private IP를 사용할 수 있습니다.

6. **VPN 설정 (옵션)**
VPC와 Lightsail 인스턴스를 VPN을 통해 연결할 수 있습니다. **OpenVPN**을 사용하여 안전한 연결을 설정하거나, **AWS Site-to-Site VPN**을 사용하여 온프레미스 네트워크와 연결할 수 있습니다.
