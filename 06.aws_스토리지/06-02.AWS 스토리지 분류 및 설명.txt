
1. 컴퓨팅 스토리지 (Compute-Connected Storage)
-------------------------------------------------
컴퓨팅 자원과 밀접하게 연결되어 있으며, 인스턴스와 데이터 처리를 중심으로 작동.

- Amazon EBS (Elastic Block Store)
  - EC2 인스턴스에 연결되는 블록 스토리지.
  - 고속 액세스와 안정적인 성능 제공.
  - 유스케이스: 데이터베이스, 파일 시스템, 애플리케이션 스토리지.

- Instance Store
  - EC2 인스턴스에 직접 연결된 임시 저장소.
  - 인스턴스 종료 시 데이터 소멸.
  - 유스케이스: 캐시 데이터 저장, 임시 데이터 처리.

- Amazon EFS (Elastic File System)
  - EC2 인스턴스 간 공유 파일 시스템.
  - 자동 확장 및 축소 지원.
  - 유스케이스: 여러 인스턴스 간 데이터 공유, 네트워크 파일 스토리지.

2. 클라우드 스토리지 (Cloud-Native Storage)
-------------------------------------------
컴퓨팅 자원과 독립적으로 동작하며, 데이터의 저장, 배포, 백업에 초점.

- Amazon S3 (Simple Storage Service)
  - 객체 스토리지로 파일 단위 데이터를 저장.
  - 무제한 확장 가능, 높은 내구성과 가용성.
  - 유스케이스: 백업 및 복구, 정적 웹사이트 호스팅, 빅데이터 분석.

- Amazon Glacier
  - 장기 보관용 저비용 스토리지.
  - 접근 속도 느리지만 보관 비용이 매우 저렴.
  - 유스케이스: 데이터 아카이빙, 규정 준수 데이터 저장.

- Amazon FSx
  - 고성능 파일 시스템 서비스.
  - Windows File Server와 Lustre 기반 지원.
  - 유스케이스: 데이터 분석, 고성능 컴퓨팅(HPC), 파일 공유.

- AWS Backup
  - 다양한 스토리지의 백업 관리 서비스.
  - 자동 백업 및 복구 제공.
  - 유스케이스: 규정 준수, 데이터 보호.

3. 하이브리드 스토리지 (Hybrid Storage)
---------------------------------------
온프레미스 데이터센터와 클라우드 스토리지를 연결하여 사용.

- AWS Storage Gateway
  - 온프레미스와 클라우드 간 데이터를 연결.
  - 파일, 볼륨, 테이프 게이트웨이 옵션 제공.
  - 유스케이스: 온프레미스 백업, 클라우드로의 데이터 아카이빙.

- AWS Outposts
  - AWS 서비스를 온프레미스 데이터센터에 제공.
  - 클라우드와 로컬 환경 통합.
  - 유스케이스: 저지연 애플리케이션, 데이터 로컬화.

4. 데이터 전송 스토리지 (Data Transfer Tools)
----------------------------------------------
AWS 외부 데이터나 온프레미스 데이터를 클라우드로 이동.

- AWS Snowball
  - 물리적 데이터 전송 장치.
  - 유스케이스: 대규모 데이터 이전, 오프라인 전송.

- AWS Snowmobile
  - 엑사바이트 수준의 대규모 데이터 전송 장치.
  - 유스케이스: 전체 데이터센터 이전.

- AWS DataSync
  - 클라우드와 온프레미스 간 데이터 전송 자동화.
  - 유스케이스: 지속적인 데이터 이동, 온프레미스에서 클라우드로 백업.
